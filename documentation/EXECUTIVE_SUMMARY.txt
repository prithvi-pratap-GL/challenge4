================================================================================
SYSTEM ARCHITECTURE
================================================================================

Workflow Flow:
  1. User submits startup analysis request
  2. System creates unique analysis ID
  3. Orchestrator runs workflow:
     - Bull Agent (investment case)
     - Bear Agent (rejection case)
     - Red Team Agent (fact-checking)
     [parallel, then converge]
     - Reviewer Agent (quality gate)
     [optional retry]
     - Committee Agent (decision)
     - Digital Twin Agent (scenarios)
     - Final Report Generation
  4. Results stored and returned to user

Data Pipeline:
  StartupInput -> Agents -> FinalReport
  All intermediate outputs preserved in AnalysisState
  Complete audit trail in database
